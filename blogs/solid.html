SOLID

Single Responsibility Principle

    Nguyên lý đơn trách nhiệm: mỗi class chỉ nên giữ 1 trách nhiệm (chức năng) duy nhất (chỉ có thể thay đổi class vì 1 lí do duy nhất )

Open/Closed principle

    Có thể thoải mái mở rộng 1 module nhưng hạn chế sửa đổi có sẵn trong module đó (open for extension but closed for modification)

Liskov Substitution Principle

    Nguyên lý thay thế Liskov
    Trong 1 chương trình, các object của class con có thể làm thay đổi object của class cha nhưng k làm thay đổi tính đúng đắn của chương trình 
    2 VD vi phạm LSP:
    Chim cánh cụt là chim
    Hình vuông là hình chữ nhật 
    => tạo class cha Shape, Square và Rectangle thừa kế Shape
    Class A kế thừa Class B, chỉ khi Class A có thể thay thế Class B
    VD: foreach có thể chạy cho List, ArrayList, LinkedList,… vì List, ArayList có thể thay thế cho IEnumerable

Interface Segregation Principle

    Nguyên lý phân tách Interface
    Interface là 1 lớp rỗng chứa nhiều phương thức không đc hiện thực, không đc khai bao thuộc tính. Các Class hiện thực Interface phải hiện thực tất cả các phương thức có trong Interface
    => để thiết kế 1 hệ thống linh hoạt, dễ thay đổi. Các module có thể gọi các hàm chức năng của nhau thông qa Interface mà không cần quan tâm hiện thực bên dưới
    Nguyên lý này là cần chia nhỏ các Interface tránh việc hiện thực các method không cần thiết

Dependency Inversion Principle

    Nguyên lý đảo ngược phụ thuộc
      - Các module cấp cao ko nên phụ thuộc các module cấp thấp. Cả 2 nên phụ thuộc vào Interface (Abstraction)
      - Các Interface (Abstraction) ko nên phụ thuộc vào chi tiết mà là ngược lại. (Các Class giao tiếp với nhau thông qua Interface chứ ko phải là Implementation) 

    VD:Ở đây, module cấp cao chính là ổ điện, interface chính là đuôi đèn tròn, 2 module cấp thấp là bóng đèn tròn và bóng đèn huỳnh quang.
    Các Module XMLDatabase, SQLDatabase phải implement IDatabase
    IDatabase db = new XMLDatabase();
    IDatabase db = new SQLDatabase();
    IDatabase db = new Database()
    db.Save(orderId);
